# Movie-Recommendation-System
Movie Recommendation System (K-Means Clustering) This project applies K-Means clustering to group similar movies based on genre and numerical attributes, enabling meaningful movie recommendations without using labeled data.

# Overview
The system analyzes a dataset of 5,562 movies containing metadata such as genre, rating, runtime, year, and more. Instead of using user ratings history (collaborative filtering), this project focuses on content-based clustering, where movies are grouped according to shared characteristics. Once clusters are formed, recommendations are generated by suggesting movies from the same cluster as the selected film.

# Problem Goal
To build a movie recommendation system that: • Identifies similarity between movies • Groups movies into meaningful clusters • Recommends movies based on cluster membership • Uses unsupervised learning (no labeled data required)

This task is formalized as a clustering problem using K-Means.

# Dataset
Dataset: IMDB_All_Genres_etf_clean1.csv Size: 5,562 movies Attributes: 10 columns

Features Used for Clustering: • Main Genre • Side Genre • Rating • Runtime (Minutes)

The dataset contains: • Numerical attributes (Rating, Runtime, Year) • Categorical/text attributes (Genres, Title, Director, etc.)

No missing values were found in the dataset.

# Methodology
Data Mining Formalization • Task Type: Unsupervised Learning • Technique: K-Means Clustering

Feature Engineering • Combined text features (Title + Genres) • Converted text to numeric vectors using CountVectorizer • Scaled numerical features using MinMaxScaler • Combined text + numeric features into final matrix X

Model Training • Algorithm: KMeans • n_clusters = 180 • random_state = 42 • n_init = 10

The Elbow Method was applied to analyze WCSS values and determine cluster size.
